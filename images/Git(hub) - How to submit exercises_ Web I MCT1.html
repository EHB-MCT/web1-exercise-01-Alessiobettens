    Git(hub) - How to submit exercises

    Join the classroom & accept the assignment

    To submit an exercise, you'll need an invitation URL to pull the source code from Github.

    For example, this is the invitation URL for exercise-01:
    https://classroom.github.com/a/qbtwkbOR (Koppelingen naar een externe site.)

    Open the URL in the browser.

    Click Authorize GitHub Classroom.

    Look for your name on the list.

    Click on the button Accept this assignment.

    You will see the screen below.

    Wait a moment and then refresh the webpage.

    The repository (short: repo) of the exercise was duplicated as a unique folder with your name as a suffix. A link was made between your Github account and the EHB-MCT classroom.

    Click on the URL to go to the repo.

    The repo opens.

    Note that an index.html file has already been added to the repo.

     

    Git Clone (to local repo)

    We now want to download the repository locally on our computer so that we can modify the code. We do this by cloning the repo.

    Click the Code button and then copy the URL of your repo.

    Open Visual Studio Code.

    In the menu, click View - Command Palette... or use the keyboard shortcut Ctrl + Shift + P.

    Type git clone and press Enter.

    Paste the URL of the remote repo and press Enter.

    An explorer window will open. Now we have to choose where we want to place the local folder on our computer.

    Navigate to the folder multimedia/web1.

    Click the Select repository Location button.

    A pop-up screen will appear at the bottom left.

    Click the button Open.

    Note that our repo was opened in Visual Studio Code.

    When we look at the web1 folder we will see that the new folder has been added.

    An invisible .git folder was automatically added to this folder.

    By default, invisible folders are not displayed. It is important that the folder .git is left unchanged.

     

    Add, Commit & Push

    We're going to make a change to the source code. Then we're going to push the files back to GitHub.

    In this case, we will remove the first line of code.

    Then we'll add some HTML elements.

    Press Ctrl + S to save the file.

    Click on the tab Source Control.

    The M next to the file means this file has been modified.

    Click on the plus sign next to Changes to add all the modified files.

    The file was moved from Changes to Staged Changes. We have now staged our files and we're ready to commit.

    Type a meaningful message explaining the changes to your code.

    Press the V button to make a commit.

    The last step is to push the code to GitHub.

    Click on the three dots and choose Push from the drop-down menu.

    Check your online repo and see if the changes were made.
